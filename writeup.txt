3. CUSTOM TEST CASES
=====================

Example 1: Type Mismatch Detection
-----------------------------------
File: test_type_mismatch.mC

Source Code:
int main() {
    int x;
    char c;
    x = c;
}

Purpose:
Tests that the semantic analyzer detects type mismatches in assignment 
statements. An integer variable is assigned a character value, which 
should trigger a type mismatch error.

Expected Output:
error: line 4: Type mismatch in assignment.
Compilation finished.

Testing Command:
./obj/mcc test_type_mismatch.mC


Example 2: Array Constant Expression Evaluation
-----------------------------------------------
File: test/custom_array_const.mC

Source Code:
char main() {
    int arr[5];
    arr[2 + 4] = 1;
    return 0;
}

Purpose:
Tests that the semantic analyzer can evaluate constant expressions at 
compile time for array indexing. The expression "2 + 4" evaluates to 6, 
which exceeds the array bound of 5. This validates the constant expression 
evaluator (eval_const_int_expr function) correctly computes arithmetic 
expressions and the array bounds checker properly detects out-of-bounds access.

Expected Output:
error: line 3: Statically sized array indexed with constant, out-of-bounds expression.
Compilation finished.

Testing Command:
./obj/mcc test/custom_array_const.mC
